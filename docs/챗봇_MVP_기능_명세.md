# AI 여행 추천 MVP 기능 및 API 명세

> 상태: 현재 `master` 구현 기준 As-Is 명세  
> 대상 화면: `/chatbot`  
> 연동 API: `GET /api/places`, `POST /api/chat`

## 1. 목적과 범위

사용자가 서울 지역의 숙소를 하나 선택하면 숙소에서 가까운 장소 최대 5곳을 계산하고, 이 장소들을 근거로 AI가 하루 여행 일정을 작성합니다. 추천 장소는 채팅 화면의 목록과 지도에서 함께 확인할 수 있습니다.

현재 MVP는 자유로운 서울 관광 질의응답이나 게시글 검색을 제공하는 범용 챗봇이 아닙니다. **선택한 숙소를 기준으로 한 근거리 장소 및 하루 일정 추천**에 범위를 한정합니다.

## 2. 사용자 흐름

```text
/chatbot 진입
→ 숙소 선택하기
→ 숙소명 또는 주소 검색
→ 기준 숙소 선택
→ 빠른 질문 선택 또는 직접 질문 입력
→ 추천 요청
→ AI 답변과 거리순 장소 최대 5곳 표시
→ 추천 장소를 목록과 지도에서 확인
```

숙소를 선택하기 전에는 질문 입력과 전송 버튼을 비활성화합니다. 첫 질문을 보내기 전에는 다음 빠른 질문을 제공합니다.

* 숙소 주변 관광지 추천해줘
* 하루 여행 코스 추천해줘
* 가까운 명소 알려줘

## 3. 화면 구성

### 3.1 숙소 선택

* 모달에서 서울 지역의 `숙박` 카테고리만 조회합니다.
* 숙소명 또는 주소 키워드 검색을 지원합니다.
* 검색어 입력 후 300ms가 지나면 자동 조회합니다.
* 한 페이지에 6개를 표시하고 이전·다음 페이지 이동을 제공합니다.
* 각 항목에는 썸네일, 카테고리, 이름, 주소를 표시합니다.
* 숙소를 선택하면 모달을 닫고 선택한 숙소 카드를 표시합니다.

### 3.2 채팅

* 사용자 메시지와 AI 답변을 시간순으로 표시합니다.
* 요청 중에는 입력창과 전송 버튼을 비활성화하고 응답 대기 애니메이션을 표시합니다.
* API 요청 실패 시 오류 문구와 `다시 시도` 버튼을 표시합니다.
* 마지막 응답의 추천 장소가 있으면 이름, 카테고리, 숙소와의 직선거리(km)를 목록으로 표시합니다.
* 새 응답이 도착하면 이전 추천 장소 목록을 새 결과로 교체합니다.

### 3.3 지도

* 첫 질문 전송 후 데스크톱에서는 채팅 오른쪽, 820px 이하에서는 채팅 아래에 표시합니다.
* 기준 숙소는 `H`, 추천 장소는 거리순 `1`~`5` 마커로 표시합니다.
* 숙소와 각 추천 장소 사이에 점선 직선을 표시합니다.
* 모든 좌표가 보이도록 지도 영역을 자동 조정합니다.
* 지도 마커를 클릭하면 장소명과 거리를 팝업으로 표시하고 채팅 목록의 같은 장소를 활성화합니다.

## 4. 추천 처리 규칙

1. `accommodation_id`로 `서울_숙박.json`에서 기준 숙소를 조회합니다.
2. `서울_관광지.json`, `서울_레포츠.json`, `서울_문화시설.json`에서 좌표가 있는 장소를 읽습니다.
3. Haversine 공식으로 숙소와 각 장소의 직선거리를 계산합니다.
4. 거리 오름차순으로 정렬해 최대 5곳을 선택합니다.
5. 숙소 정보, 추천 장소 5곳, 사용자 질문을 AI 프롬프트에 포함합니다.
6. AI는 제공된 장소만 사용해 시간 순서가 있는 하루 여행 일정을 작성합니다.

추천 장소 선정은 AI가 아니라 서버의 거리 계산으로 결정합니다. 현재는 사용자 질문의 장소 유형이나 조건에 따라 후보를 다시 필터링하지 않습니다.

## 5. API 명세

### 5.1 기준 숙소 검색

```http
GET /api/places?keyword={keyword}&category=숙박&region=서울&page=1&size=6
```

장소 API의 기존 페이지 응답을 그대로 사용합니다. 프런트에서 사용하는 주요 필드는 다음과 같습니다.

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `items[].id` | string | 장소 복합 ID. 프런트는 끝의 숫자를 챗봇 요청 ID로 변환합니다. |
| `items[].name` | string | 숙소명 |
| `items[].category` | string | 장소 유형. 이 화면에서는 `숙박` |
| `items[].address` | string \| null | 주소 |
| `items[].image_url` | string \| null | 썸네일 URL |
| `items[].latitude` | number \| null | 위도 |
| `items[].longitude` | number \| null | 경도 |
| `page` | integer | 현재 페이지 |
| `total_pages` | integer | 전체 페이지 수 |

### 5.2 AI 여행 추천

```http
POST /api/chat
Content-Type: application/json
```

요청 본문:

```json
{
  "message": "하루 여행 코스 추천해줘",
  "accommodation_id": 3056929,
  "history": []
}
```

| 필드 | 타입 | 필수 | 현재 동작 |
| --- | --- | --- | --- |
| `message` | string | 예 | 사용자의 현재 질문 |
| `accommodation_id` | integer | 예 | 숙박 JSON의 숫자형 `contentid` |
| `history` | array | 아니오 | 요청 스키마에는 있으나 현재 기능에서는 빈 배열만 전송하고 서버가 사용하지 않음 |

성공 응답:

```json
{
  "answer": "숙소에서 출발해 가까운 장소부터 방문하는 하루 일정을 추천합니다.",
  "places": [
    {
      "id": "2733967",
      "name": "추천 장소명",
      "category": "A02",
      "latitude": 37.567,
      "longitude": 126.978,
      "distance": 1.24
    }
  ]
}
```

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `answer` | string | AI가 작성한 한국어 여행 일정 또는 서버 상태 안내 문구 |
| `places` | array | 숙소에서 가까운 장소 최대 5곳 |
| `places[].id` | string | 원본 관광 데이터의 `contentid` |
| `places[].name` | string | 장소명 |
| `places[].category` | string \| null | 원본 데이터의 대분류 코드 또는 기본 분류 |
| `places[].latitude` | number \| null | 위도 |
| `places[].longitude` | number \| null | 경도 |
| `places[].distance` | number \| null | 숙소와의 직선거리(km), 소수 둘째 자리 반올림 |

## 6. 상태 및 예외 처리

| 상황 | 현재 동작 |
| --- | --- |
| 숙소 미선택 | 입력창과 전송 버튼 비활성화 |
| 잘못된 숙소 ID | 프런트에서 요청하지 않고 오류 문구 표시 |
| 숙소를 찾을 수 없음 | HTTP 200과 `숙소 정보를 찾을 수 없습니다.` 답변, 빈 장소 배열 반환 |
| OpenAI 키 미설정 | HTTP 200과 `현재 AI 추천 기능을 사용할 수 없습니다.` 답변 반환 |
| AI 응답 생성 실패 | HTTP 200과 오류 안내 답변 반환 |
| HTTP 또는 네트워크 오류 | 프런트 오류 문구와 재시도 버튼 표시 |
| 컴포넌트 종료 | 진행 중인 프런트 요청 취소 |

## 7. 현재 MVP 제외 범위

* 후속 질문의 대화 문맥 유지
* 대화 내역의 서버·브라우저 저장
* 새 대화 생성 및 대화 목록
* 쇼핑·여행코스·축제 데이터를 포함한 추천
* 커뮤니티 게시글 자연어 검색
* 사용자 질문에 따른 장소 후보 필터링 또는 재정렬
* 이동 시간·영업시간·교통수단을 반영한 실제 경로 최적화
* 스트리밍 응답과 응답 중단 버튼
* 회원별 사용량 제한 및 추천 이력 관리

## 8. 구현상 주의사항

* 장소 API의 복합 문자열 ID와 챗봇 API의 숫자 `contentid`가 서로 다릅니다. 프런트는 현재 복합 ID 끝의 숫자를 추출해 전송합니다.
* 숙소 변경 시 추천 장소와 지도 선택 상태만 초기화하고, 기존 메시지는 유지합니다.
* 추천 장소 목록 선택은 활성 스타일만 변경합니다. 현재는 해당 지도 마커의 팝업을 자동으로 열거나 지도를 이동하지 않습니다.
* 현재 `history`는 동작하지 않으므로 클라이언트가 문맥 유지 기능을 전제로 사용하면 안 됩니다.
