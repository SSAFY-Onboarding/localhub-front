# AI 여행 추천 MVP 기능 및 API 명세

> **상태**: 현재 프런트엔드·백엔드 `master` 구현 기준 As-Is 명세
>
> **대상 화면**: `/chatbot`
>
> **연동 API**: `GET /api/places`, `POST /api/chat`

## 1. 목적과 범위

사용자가 자연어로 서울의 여행 장소를 질문하면 서버가 질문의 지역·의도·장소 유형을 판별하고, 실제 장소 데이터에서 최대 8곳을 선택해 AI 답변과 지도 정보를 반환한다.

숙소 선택은 선택 사항이다. 숙소를 선택하면 숙소를 추천 기준점으로 사용할 수 있고, 숙소를 선택하지 않아도 `성수에서 전시와 쇼핑 장소를 추천해줘`처럼 질문에 지역을 포함해 이용할 수 있다.

현재 기능은 장소 데이터에 근거한 추천 챗봇이며, 커뮤니티 게시글 검색이나 실제 이동 경로 최적화까지 수행하는 범용 여행 비서는 아니다.

## 2. 사용자 흐름

```text
/chatbot 진입
→ (선택) 서울 숙소 검색·선택
→ 빠른 질문 선택 또는 직접 질문 입력
→ 서버가 질문 의도·카테고리·지역 기준점 판별
→ 실제 장소 데이터에서 추천 후보 검색·정렬
→ AI 답변과 추천 장소 최대 8곳 표시
→ 목록 선택 또는 지도 마커 클릭으로 장소 확인
```

숙소를 선택하지 않아도 질문을 보낼 수 있다. 질문에 지역이 없고 기준점도 정할 수 없는 경우에는 서버가 지역을 더 구체적으로 입력하도록 안내한다.

첫 질문 전 빠른 질문 예시는 다음과 같다.

- 서울역 근처 축제를 추천해줘
- 성수에서 전시와 쇼핑 장소를 추천해줘
- 한강 주변 체험 장소를 알려줘

## 3. 화면 구성

### 3.1 숙소 선택

- 숙소 선택은 필수가 아니다.
- 모달에서 `GET /api/places`로 서울의 `숙박` 카테고리를 조회한다.
- 숙소명 또는 주소 키워드 검색, 6개 단위 페이지네이션을 지원한다.
- 각 항목에는 썸네일, 카테고리, 이름, 주소를 표시한다.
- 선택한 숙소는 카드로 표시하며 변경할 수 있다.
- 챗봇 요청에는 장소 API가 반환한 복합 문자열 ID를 그대로 보낸다.

### 3.2 채팅

- 사용자 메시지와 AI 답변을 시간순으로 표시한다.
- 이전 대화 내역을 `history`로 함께 전송해 후속 답변 생성에 사용한다.
- 요청 중에는 입력창과 전송 버튼을 비활성화하고 응답 대기 상태를 표시한다.
- 요청 실패 시 오류 문구와 다시 시도 동작을 제공한다.
- 마지막 응답의 추천 장소를 최대 8개까지 목록으로 표시한다.
- 새 추천 응답이 도착하면 장소 목록과 지도 결과를 새 응답 기준으로 교체한다.

### 3.3 지도

- 추천 장소가 있으면 데스크톱에서는 채팅 오른쪽, 820px 이하에서는 채팅 아래에 표시한다.
- 서버가 반환한 `center`를 기준점으로 사용하고 추천 장소 마커를 함께 표시한다.
- 거리 정보가 있는 경우 목록과 팝업에 직선거리(km)를 표시한다.
- 목록에서 장소를 선택하면 해당 마커로 이동해 팝업을 열고, 마커를 선택하면 목록의 같은 장소를 활성화한다.
- 추천 장소가 모두 보이도록 지도 영역을 조정한다.

## 4. 서버 추천 처리 규칙

1. `accommodation_id`가 있으면 공통 장소 서비스에서 기준 숙소를 조회한다.
2. 현재 질문을 분석해 의도, 장소 카테고리, 지역 키워드, 일반 키워드, 숙소 기준 사용 여부를 판별한다.
3. 질문에 지역이 있으면 장소 이름·주소 데이터에서 기준점을 찾고, 일부 주요 서울 지명은 내장 좌표를 보조 수단으로 사용한다.
4. 판별한 카테고리에 해당하는 장소 데이터에서 후보를 검색한다.
5. 키워드 일치도, 기준점과의 직선거리, 장소명 순으로 후보를 정렬한다.
6. 최대 8곳과 최근 대화 내역을 AI 프롬프트에 제공해 답변을 생성한다.
7. AI 답변은 서버가 찾은 장소만 근거로 작성하도록 제한한다.

질문 분류에 사용할 AI 호출이 실패하거나 API 키가 없으면 규칙 기반 분류기로 대체한다. 기준점이 없는 질문에서는 거리 값이 `null`일 수 있으며, 이 경우 키워드 일치도와 장소명을 중심으로 정렬한다.

## 5. API 명세

### 5.1 기준 숙소 검색

```http
GET /api/places?keyword={keyword}&category=숙박&region=서울&page=1&size=6
```

프런트에서 사용하는 주요 필드는 다음과 같다.

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `items[].id` | string | 장소 복합 ID, 예: `32-3056929` |
| `items[].name` | string | 숙소명 |
| `items[].category` | string | 장소 유형 |
| `items[].address` | string \| null | 주소 |
| `items[].image_url` | string \| null | 썸네일 URL |
| `items[].latitude` | number \| null | 위도 |
| `items[].longitude` | number \| null | 경도 |
| `page` | integer | 현재 페이지 |
| `total_pages` | integer | 전체 페이지 수 |

### 5.2 AI 여행 추천

```http
POST /api/chat
Content-Type: application/json
```

요청 예시:

```json
{
  "message": "성수에서 전시와 쇼핑 장소를 추천해줘",
  "accommodation_id": null,
  "history": [
    { "role": "user", "content": "서울에서 하루 여행을 하고 싶어" },
    { "role": "assistant", "content": "어느 지역을 중심으로 찾아볼까요?" }
  ]
}
```

| 필드 | 타입 | 필수 | 설명 |
| --- | --- | :---: | --- |
| `message` | string | O | 현재 질문, 1자 이상 |
| `accommodation_id` | string \| null | X | 장소 복합 ID. 숙소를 선택하지 않으면 `null` |
| `history` | array | X | 이전 사용자·AI 메시지. 기본값 `[]` |
| `history[].role` | string | O | 메시지 발화자. 프런트는 `user` 또는 `assistant` 사용 |
| `history[].content` | string | O | 이전 메시지 내용 |

성공 응답 예시:

```json
{
  "answer": "성수에서 전시와 쇼핑을 함께 즐길 수 있는 장소를 추천할게요.",
  "places": [
    {
      "id": "14-2733967",
      "name": "추천 장소명",
      "category": "문화시설",
      "latitude": 37.544,
      "longitude": 127.056,
      "distance": 0.84,
      "address": "서울특별시 성동구 ...",
      "image_url": null,
      "description": null,
      "phone": null
    }
  ],
  "center": {
    "latitude": 37.544,
    "longitude": 127.056
  },
  "query_type": "place_recommendation"
}
```

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `answer` | string | 장소 데이터에 근거한 한국어 답변 또는 상태 안내 |
| `places` | array | 추천 장소 최대 8곳 |
| `places[].id` | string | 장소 복합 ID |
| `places[].name` | string | 장소명 |
| `places[].category` | string | 장소 유형 |
| `places[].latitude` | number | 위도 |
| `places[].longitude` | number | 경도 |
| `places[].distance` | number \| null | 기준점과의 직선거리(km) |
| `places[].address` | string | 주소. 정보가 없으면 빈 문자열 |
| `places[].image_url` | string | 썸네일 URL. 정보가 없으면 빈 문자열 |
| `places[].description` | string | 장소 설명. 정보가 없으면 빈 문자열 |
| `places[].phone` | string | 전화번호. 정보가 없으면 빈 문자열 |
| `center` | object \| null | 숙소 또는 질문 속 지역에서 계산한 지도 기준점 |
| `query_type` | string | 서버가 판별한 질의 유형 |

## 6. 상태 및 예외 처리

| 상황 | 현재 동작 |
| --- | --- |
| 숙소 미선택 | 질문 입력과 전송 가능 |
| 숙소 ID가 없음 | 질문 속 지역을 기준으로 추천 시도 |
| 숙소 또는 지역 기준점을 찾지 못함 | 더 구체적인 지역 입력을 요청하는 안내와 빈 장소 목록 반환 |
| 추천 후보 없음 | 조건을 바꾸도록 안내하고 빈 장소 목록 반환 |
| 분류용 AI 호출 실패 | 규칙 기반 분류기로 대체 |
| 답변 생성용 API 키 미설정 또는 호출 실패 | 추천 장소는 유지하고 AI 답변 불가 안내 반환 |
| HTTP 또는 네트워크 오류 | 프런트 오류 문구와 재시도 동작 표시 |
| 컴포넌트 종료 | 진행 중인 프런트 요청 취소 |

## 7. 현재 MVP 제외 범위

- 대화 내역의 서버·브라우저 영구 저장과 대화 목록
- 커뮤니티 게시글 자연어 검색
- 실제 도로망, 이동 시간, 교통수단을 반영한 경로 최적화
- 영업시간, 가격, 혼잡도, 행사 일정 등 실시간 정보 보장
- 스트리밍 응답과 응답 중단 버튼
- 회원별 사용량 제한과 추천 이력 관리

## 8. 구현상 주의사항

- `accommodation_id`는 숫자 `contentid`로 변환하지 않고 장소 API의 복합 문자열 ID를 그대로 사용한다.
- 답변 생성에는 최근 대화 내역 중 최대 8개 메시지를 사용한다. 현재 질문 분류는 주로 새 메시지를 기준으로 수행한다.
- 대화와 선택 숙소는 영구 저장하지 않으므로 새로고침하거나 페이지를 벗어나면 초기화된다.
- `distance`는 실제 이동 거리가 아니라 위·경도 기반 직선거리다.
- `center`가 `null`이어도 추천 장소 좌표가 있으면 지도는 장소 범위에 맞춰 표시할 수 있어야 한다.
